# CV

A single-source CV/resume that builds into both a webpage (GitHub Pages) and a PDF (generated by GitHub Actions on every push).

## How it works

- All content lives in [`content/resume.yaml`](content/resume.yaml) — edit this to update your CV.
- `src/template.mjs` renders that data into an HTML page; `src/style.css` styles it for both screen and print.
- `scripts/build.mjs` builds `dist/index.html` from the YAML + template.
- `scripts/pdf.mjs` uses Playwright to print `dist/index.html` to `dist/cv.pdf`, so the PDF always matches the webpage exactly.
- `.github/workflows/deploy.yml` runs both build steps and deploys `dist/` to GitHub Pages on every push to `main`.

## Editing your CV

Just edit `content/resume.yaml` and push to `main` — the site and PDF rebuild and redeploy automatically.

## Local development

```sh
npm install
npm run build       # writes dist/index.html + dist/style.css
open dist/index.html  # preview in a browser (macOS)

npx playwright install chromium  # first time only
npm run build:pdf   # writes dist/cv.pdf
open dist/cv.pdf
```

Or run both build steps at once: `npm run build:all`.

## One-time GitHub Pages setup

After pushing this repo to GitHub, go to **Settings → Pages** and set **Source** to **GitHub Actions**. This can't be configured from the workflow file itself.

Once enabled, the site will be published at `https://<username>.github.io/cv/`, with the PDF downloadable at `https://<username>.github.io/cv/cv.pdf`.
